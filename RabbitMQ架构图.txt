              │   前端 (Vue.js :5173)          │

                             ▼

                    场景三: 评论异步通知
═══════════════════════════════════════════════════════════════════════════════

                   👤 用户A 发表评论
                         │
                         ▼
         ┌────────────────────────────────────┐
         │    CreateComment (Controller)      │
         │                                    │
         │  ① INSERT comment INTO MySQL       │ ← 同步完成
         │  ② DEL Redis cache                 │ ← 缓存失效
         │  ③ ctx.JSON(200, comment)          │ ← 返回给用户A
         │     ^                              │
         │     │                              │
         │  ④ publishCommentNotification()   │ ← 🔴 生产者
         │     不阻塞上面③的返回              │    (异步)
         └────────────────┬───────────────────┘
                          │
                          ▼
         ┌─────────────────────────────────────┐
         │     🟡 RABBITMQ                     │
         │     routing key = "comment.notify"  │
         │              │                      │
         │     ┌────────▼──────────────────┐   │
         │     │  Queue:                   │   │
         │     │  comment_notifications    │   │
         │     │  (持久化)                 │   │
         │     └────────┬──────────────────┘   │
         └──────────────┬─────────────────────┘
                        │
                        ▼
         ┌─────────────────────────────────────┐
         │    🟢 CONSUMER                      │
         │    consumers/notification_consumer   │
         │                                     │
         │  ① 反序列化 msg                     │
         │  ② 查文章作者                       │

         │                                     │

═══════════════════════════════════════════════════════════════════════════════
                         关键设计原则
═══════════════════════════════════════════════════════════════════════════════

  1. 同步 + 异步 混合
     ┌────────────┬─────────────────────────────────────────┐
     │ 同步完成   │ 核心业务: Redis INCR / INSERT comment   │
     │            │ 用户立即看到结果                         │
     ├────────────┼─────────────────────────────────────────┤
     │ 异步完成   │ 持久化到 MySQL / 发送通知               │
     │            │ 不影响用户体验，允许重试                  │
     └────────────┴─────────────────────────────────────────┘

  2. 消息不丢失
     ┌────────────┬─────────────────────────────────────────┐
     │ Queue      │ durable=true, 服务器重启后队列仍在      │
     │ Message    │ persistent=true, 服务器重启后消息仍在    │
     │ ACK        │ 手动确认, 消费成功才删                   │
     │ DLQ        │ 处理失败的消息进入死信队列, 不丢弃       │
     └────────────┴─────────────────────────────────────────┘

  3. 批量 vs 逐条
     ┌────────────┬─────────────────────────────────────────┐
     │ 点赞消费者 │ 批量聚合, 5秒刷一次, 减少 DB 写压力     │
     │ 通知消费者 │ 逐条处理, 尽快送达, 通知不能等           │
     └────────────┴─────────────────────────────────────────┘

  4. 断线重连
     go ReconnectRabbitMQ() 在 main.go 启动
     → 监听 NotifyClose
     → 3 秒后重试
     → 无限循环直到成功
